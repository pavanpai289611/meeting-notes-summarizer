# API Reference

This app exposes exactly one API endpoint. It's same-origin only (the Express server serves
both this endpoint and the built frontend), so there's no CORS configuration and no
cross-origin usage is supported.

This reference matches the current implementation in `server/routes/summarize.ts` and
`server/services/claudeClient.ts` — not an aspirational spec.

---

## `POST /api/summarize`

Takes raw meeting notes or a transcript and returns a structured summary generated by Claude.

### Authentication

**None.** This is a single-user, no-auth tool by design (see
[docs/product-brief-and-prd.md](./product-brief-and-prd.md) §2.9 Constraints). The endpoint
does not check for any credentials, session, or API key from the caller. The Claude API key
lives server-side only, in `ANTHROPIC_API_KEY`, and is never read from or exposed to the
client.

### Request

```
POST /api/summarize
Content-Type: application/json
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `transcript` | `string` | Yes | Must be non-empty after trimming whitespace. No explicit maximum length is enforced by the route; in practice it's bounded by Express's default JSON body-size limit (~100kb, since no explicit `limit` is configured on `express.json()`) and by Claude's own context window. |

```json
{ "transcript": "Alice and Bob discussed the Q1 roadmap. They decided to ship the beta in March. Alice will draft the announcement." }
```

### Success response — `200 OK`

```json
{
  "summary": {
    "keyDiscussionPoints": ["string", "..."],
    "decisionsMade": ["string", "..."],
    "actionItems": [
      { "task": "string", "owner": "string" }
    ]
  }
}
```

- `keyDiscussionPoints`, `decisionsMade`, `actionItems` are **always present**, even if empty
  (`[]`) — an empty array means "none identified," not "field omitted."
- `actionItems[].owner` is either the exact name mentioned in the source text, or the literal
  string `"Unassigned"` if no owner was stated. The model is instructed never to guess a name
  that isn't in the source text.
- This shape is enforced by Claude's structured-outputs feature (`output_config.format`, via a
  Zod schema in `claudeClient.ts`) — the server does not hand-parse or regex the model's
  output, so a schema-violating response from Claude surfaces as an error (see below), not a
  malformed `200`.

### Error responses — shared envelope

Every error response uses the same shape:

```json
{
  "error": {
    "code": "EMPTY_INPUT | SUMMARIZATION_FAILED | TIMEOUT | BAD_REQUEST | PAYLOAD_TOO_LARGE",
    "message": "human-readable string, safe to display directly in a UI"
  }
}
```

| HTTP status | `error.code` | When it happens |
|---|---|---|
| `400 Bad Request` | `EMPTY_INPUT` | `transcript` is missing, not a string, or empty/whitespace-only after trimming. The Claude API is never called in this case. |
| `502 Bad Gateway` | `SUMMARIZATION_FAILED` | The call to the Claude API fails for any reason other than a timeout — auth failure, rate limit, a 5xx from Claude, or the model's response failing schema validation (`parsed_output` comes back null). |
| `504 Gateway Timeout` | `TIMEOUT` | The Claude API call exceeds the server's client-side timeout (`30_000`ms, set in `claudeClient.ts`), detected via the Anthropic SDK's `APIConnectionTimeoutError`. |
| `400 Bad Request` | `BAD_REQUEST` | The request body is not valid JSON |
| `413 Payload Too Large` | `PAYLOAD_TOO_LARGE` | The request body exceeds the server's size limit |

In every error case, the real underlying error (SDK exception, stack trace, upstream error
body) is logged server-side via `console.error` only — it is never included in the JSON
response sent to the client.

### Examples

**Empty input:**
```bash
curl -X POST http://localhost:3000/api/summarize \
  -H "Content-Type: application/json" \
  -d '{"transcript": ""}'
# 400
# {"error":{"code":"EMPTY_INPUT","message":"Please paste some meeting notes before summarizing."}}
```

**Success:**
```bash
curl -X POST http://localhost:3000/api/summarize \
  -H "Content-Type: application/json" \
  -d '{"transcript": "Alice and Bob discussed the Q1 roadmap. They decided to ship the beta in March. Alice will draft the announcement."}'
# 200
# {"summary":{"keyDiscussionPoints":[...],"decisionsMade":[...],"actionItems":[{"task":"Draft the announcement","owner":"Alice"}]}}
```

**Oversized payload:**
```bash
curl -X POST http://localhost:3000/api/summarize \
  -H "Content-Type: application/json" \
  -d "{\"transcript\": \"$(head -c 150000 </dev/zero | tr '\0' 'x')\"}"
# 413
# {"error":{"code":"PAYLOAD_TOO_LARGE","message":"Meeting notes are too long. Please shorten and try again."}}
```

### Notes

- No rate limiting, request throttling, or usage caps are implemented on this endpoint —
  explicitly out of scope for v1 (see PRD §2.6).
- Nothing sent to this endpoint is persisted server-side. All persistence is client-side
  `localStorage`, keyed `mns:v1:summaries` (see the architecture doc's Data Model section).
